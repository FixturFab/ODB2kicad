  - Links DRC test provider .o files from pcbnew_kiface_objects (self-registering)
  - Uses --start-group/--end-group for circular static library dependencies
  - Matches KiCad build compile definitions (KICAD_IPC_API, PCBNEW, USINGZ, etc.)

  - Initializes wxWidgets (required by KiCad libraries)
  - Reads PCB file from disk, calls API, prints JSON results
- Task 4.6: Built and tested - produces 4 DRC violations (silkscreen warnings)

### Key findings and bugs fixed:
- DRC test providers use static self-registration and must be explicitly linked as .o files
  from pcbnew_kiface_objects (not in libpcbcommon.a)
- Circular link dependencies between KiCad static libraries require --start-group/--end-group
- KICAD_IPC_API define is critical: changes PGM_BASE class layout (adds 2 unique_ptr members)
  Without it, heap corruption occurs due to class layout mismatch with pre-built libraries
- All compile definitions must match KiCad's build (PCBNEW, USINGZ, WXUSINGDLL, __WXGTK__, etc.)
- PGM_BASE thread pool must be initialized: KICAD_SINGLETON::Init() is not exported from
  libkicommon.so, so we directly allocate BS::priority_thread_pool on m_singleton.m_ThreadPool
  Without it, GetKiCadThreadPool() dereferences null m_ThreadPool via PGM_BASE::GetThreadPool()
- PGM_BASE is required by GAL (graphics abstraction layer) which is transitively needed by
  some DRC test providers. Without Pgm(), GAL::GAL() asserts.
- InitEngine(wxFileName()) must be called before RunTests() to register test providers
- testFootprints=false skips schematic parity checks (we have no schematic)
- Output matches kicad-cli format: same JSON schema, same violation types
- kicad-cli reports 6 violations (includes 2 library warnings), our lib reports 4
  (silkscreen-only, since library checks require schematic data)
